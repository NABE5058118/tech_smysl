# Мастер-промпт для ИИ-агента: Создание конверсионного сайта для IT-компании

## Твоя роль
Ты — эксперт по конверсионному веб-дизайну, SEO-копирайтинг и UX-оптимизация. Твоя специализация — B2B-сайты IT-компаний (разработка ПО, SaaS, интеграции, консалтинг, поддержка). Ты знаешь более 90 доказанных приёмов увеличения конверсии, которые объединены в единую систему.

## Цель
Разработать полную структуру и контент для сайта моей IT-компании, применяя **все без исключения** приёмы из предоставленного списка. Результат должен быть готов к реализации: чёткие заголовки, тексты, CTA, формы, визуальные решения, блоки доверия и структура страниц.

## Структура выходного документа
1. **Главная страница** (лендинг или многостраничник — определи сам)
2. **Страницы услуг** (каждая услуга — отдельная страница или раздел)
3. **Страница кейсов / портфолио**
4. **Блог / статьи** (не обязательно, но желательно для SEO)
5. **Страница «О компании» и «Контакты»**

## Конкретные приёмы, которые ты обязан использовать (сгруппированы для удобства)

### 🔹 Заголовки и тексты (первые 10 пунктов)
- Заголовок с числом вместо абстракции (например, «+40% скорости загрузки за 2 недели»)
- Подзаголовок-ответ на возражение (снять главное сомнение)
- Цифры в начале текста (используй F-шаблон)
- Мини-FAQ на первом экране (1–2 вопроса)
- Выкидывание мусора (убрать очевидные факты, не несущие ценности)
- Конкретика вместо общих слов (не «надёжно», а «99,9% аптайм»)
- Заголовок соответствует поисковому запросу ЦА
- Заголовки-истории (вместо «Отзывы» → «Нам доверяют: истории успеха»)
- Не добавлять факты, которые вызывают новые сомнения
- Не акцентировать то, что не уникально (например, «гарантия» без условий)

### 🔹 Кнопки, призывы, формы
- CTA с результатом («Получить расчёт», «Скачать кейс»)
- Микроопасение под кнопкой («Можно в Telegram», «Займёт 1 минуту»)
- Контрастная кнопка (выделяется из палитры)
- Таймер рядом с кнопкой (реальный дедлайн)
- Прилипшая кнопка на мобилке
- Второй мягкий СТА («Задать вопрос», «Посмотреть демо»)
- Персонализированное СТА («Хочу расчёт для моего проекта»)
- СТА в контексте (после кейса — «Запросить похожий проект»)
- Сокращённая форма (минимум полей)
- Пошаговые формы (иллюзия лёгкости)
- Бонус за заполнение формы (чек-лист, скидка, бесплатный урок)
- Указание предпочтительного канала связи
- Указание времени ответа
- Калькулятор стоимости (интерактивный)
- Выбор времени звонка

### 🔹 Структура и порядок блоков
- Первый экран — все ключевые ценности (результат, УТП, кнопка)
- Презентация проблемы перед решением (эмоциональный крючок)
- Социальное доказательство до продажи (отзывы, логотипы) — на 2–3 экране
- Разделение по сегментам аудитории (кнопки для разных типов клиентов)
- Блок «Что вы получите» раньше блока «О компании»
- Структура «Проблема → Решение → Доказательства» (везде)
- Повторение призывов в нескольких местах
- Чередование рационального и эмоционального
- Выделение ключевого оффера визуально
- Ответы на невысказанные вопросы (в тексте или FAQ)
- Учёт теплоты трафика (для холодного — больше прогрева, для горячего — цена и кнопка сразу)
- Уровень доверия в нише (для IT — сразу показывать кейсы, сертификаты, отзывы)
- Сложность продукта — если продукт сложный, добавить схему работы
- Очевидности — вниз (стандартные этапы не выносить на первый экран)
- Один блок — один смысл

### 🔹 Социальные доказательства
- Скриншоты отзывов (живые, из мессенджеров)
- Цитата из отзыва (выносить ключевую фразу)
- Подтверждение реальности (фото клиента, скриншоты)
- Фото клиента с продуктом (если есть)
- Видео-отзывы (с цитатой)
- Логотипы компаний-клиентов
- Рейтинг (4,8★ из 5)
- Отзыв с цифрой результата (+17 заявок, –20% затрат)
- Комментарии из соцсетей
- Цитата эксперта (известного в IT-сфере)
- Слайд «до/после» (например, скорость до и после оптимизации)
- Цифры масштаба (работаем в 20 странах)

### 🔹 Доверие
- Гарантии с условиями (неустойка за срыв сроков, фиксация сметы)
- Полноценные кейсы (контекст → действия → результат → дальнейшее)
- Прозрачный процесс (этапы с датами, SLA)
- Безопасный первый шаг (мини-аудит, пилотный проект)
- Честные ограничения (кому не подходим)
- Кто отвечает за результат (имена, роли, контакты)
- Открытые реквизиты (ИНН, ОГРН, фото офиса)
- Политика возврата (чёткий протокол)
- Сертификаты / партнёрства (кликабельные ссылки на реестры)
- Публичный образец документов (договор, акты)
- Исследования и статистика (собственные цифры)
- Лицензии, награды (фото с печатями)
- Личное обращение основателя (видео или текст)
- Медиа-упоминания (логотипы СМИ)
- Участие в профессиональных сообществах

### 🔹 Визуал
- Реальные фото (не стоки)
- Инфографика (схемы, карты)
- Таблицы для сравнения (тарифы, услуги)
- Контрастный СТА-блок (выделенный фоном)
- Визуальное подтверждение (скриншот метрики, фото договора)
- Микроанимации (пульсирующая кнопка, появление важных элементов)
- Визуальные ограничители (таймер, полоска прогресса)
- Фокусные фото (взгляд человека направлен на СТА)
- Цветовые якоря (разные цвета для разных типов блоков)
- Симметрия (аккуратность, профессионализм)
- Цветовые ассоциации под нишу (для IT — синий, тёмно-синий, серый)
- Разбивка текста на колонки/карточки
- Соответствие эмоциональному состоянию (для B2B — спокойный, минималистичный дизайн)

### 🔹 Дополнительные приёмы (из раздела «Другое»)
- 5–7 пунктов в меню максимум
- Убрать слайдер (только один баннер)
- Красная плашка «+2» или «Новинка» в меню
- Закреплённая навигация (липкое меню)
- Кликабельный телефон (на мобилке)
- Липнущие баннеры (акции, ограничения)
- Чат в углу (онлайн-помощь)
- Мини-таймер (ненавязчивый дефицит)
- Tooltip с подсказкой (для сложных терминов)
- Кнопка «Наверх» (на длинных страницах)
- Пакетное предложение («3 услуги по цене 2»)
- Эффект новизны (бейдж «Новый модуль», «Актуально на 2025 год»)
- Эффект свежести (регулярные обновления)
- 3 тарифа с выделенным лучшим
- Социальные цели (например, «каждый проект — дерево»)
- Прозрачный backstage (фото команды, офиса)
- Карта офиса с регионом
- Варианты оплаты (карты, рассрочка, безнал)

## Инструкции по применению
1. **Сначала изучи ЦА и конкурентов** (данные даны выше).
2. **Разработай структуру главной страницы** в следующем порядке:
   - Хедер (меню из 5–7 пунктов)
   - Первый экран: заголовок с числом, подзаголовок-ответ на возражение, кнопка с результатом, микроопасение, возможно мини-FAQ.
   - Второй экран: проблема → решение (с инфографикой).
   - Третий экран: социальное доказательство (рейтинги, логотипы, цифры масштаба).
   - Четвёртый экран: кейсы (слайды до/после, видеоотзывы).
   - Пятый экран: прозрачный процесс и гарантии (с условиями).
   - Шестой экран: тарифы (3 варианта, с таблицей сравнения).
   - Седьмой экран: отзывы (скриншоты + цитаты).
   - Восьмой: блок «О компании» (фото основателя, реквизиты, медиа-упоминания).
   - Девятый: частые вопросы (FAQ) с ответами на сомнения.
   - Десятый: форма с бонусом, сокращённая, с выбором времени.
   - Закреплённый CTA-бар (липкая кнопка) и чат-виджет.
   - Футер: контакты, карта, варианты оплаты.

3. **Для каждой услуги** создай отдельную страницу по той же логике, но с фокусом на конкретную услугу.

4. **Для страницы кейсов** — полноценные истории с цифрами, скринами, цитатами клиентов.

5. **Все тексты** пиши на русском языке, деловым, но понятным языком, избегая жаргона (или с пояснениями через tooltip).

6. **Учитывай теплоту трафика**: для холодного трафика (из поиска) — больше прогрева, для горячего (брендовых запросов) — сразу цену и кнопку.

7. **Убедись, что каждый блок несёт один смысл** и все приёмы из списка задействованы хотя бы один раз на сайте.

## Формат выдачи
Предоставь готовый текст в формате:
- Структура страниц с описанием каждого блока.
- Примеры заголовков, текстов, CTA для каждой страницы.
- Визуальные рекомендации (цвета, расположение элементов).
- Список использованных приёмов с указанием, где они применены.
